# 黑洞 BI 数据加密说明

## 加密算法

**AES-256-GCM** (高级加密标准)

- 密钥长度：256 位 (32 字节)
- 模式：GCM (提供认证和完整性保护)
- IV：12 字节随机数 (每次加密不同)
- AuthTag：16 字节 (验证数据完整性)

---

## 文件结构

### 加密文件 (`data/*.sc`)

```
格式：Base64(IV[12] + AuthTag[16] + 密文)
```

**示例：**
```
6Yx3mK8pQr... (Base64 字符串，完全不可读)
```

### 解密后内容

```json
{
  "generatedAt": "2026-03-07T09:49:05.287Z",
  "range": { "start": "2026-02-05", "end": "2026-03-06" },
  "regions": [
    {
      "region": "韩国",
      "currency": "USD",
      "yesterday": { "dau": 1063, "arpu": 2.28 }
    },
    ...
  ]
}
```

## 使用方法

### 1. 获取密钥

联系数据管理员获取 `BI_ENCRYPT_KEY` (32 字节 hex 字符串)

### 2. 设置密钥

**方式 A: 环境变量**
```bash
export BI_ENCRYPT_KEY=你的密钥 (联系数据管理员获取)
```

**方式 B: .env.enc 文件**
```bash
# 复制示例文件
cp .env.enc.example .env.enc

# 编辑填入真实密钥
nano .env.enc
```

### 3. 解密数据

```bash
# 输出到终端
node decrypt.js data/20260307.sc

# 保存到文件
node decrypt.js data/20260307.sc data/20260307.json
```

### 4. 查看解密后的数据

```bash
cat data/20260307.json
```

## 安全说明

### ✅ 安全做法

- 密钥存储在本地环境变量
- `.env.enc` 文件加入 `.gitignore`
- 通过安全渠道分享密钥 (如 Signal/面对面)
- 定期轮换密钥

### ❌ 危险做法

- 密钥硬编码在脚本中
- 密钥提交到 Git
- 密钥通过明文邮件/聊天发送
- 多人共享同一密钥 (无法追溯泄露源)

## 密钥管理

### 生成新密钥

```bash
node -e "console.log(crypto.randomBytes(32).toString('hex'))"
```

### 轮换密钥流程

1. 生成新密钥
2. 用旧密钥解密所有 `.sc` 文件
3. 用新密钥重新加密
4. 提交新加密文件
5. 安全分发新密钥给授权用户
6. 销毁旧密钥

## 故障排查

### 错误：未找到加密密钥

```
❌ 解密失败：未找到加密密钥
```

**解决**: 设置 `BI_ENCRYPT_KEY` 环境变量或创建 `.env.enc` 文件

### 错误：密钥格式错误

```
❌ 解密失败：错误的密钥长度
```

**解决**: 密钥必须是 32 字节 hex 字符串 (64 个字符)

### 错误：认证失败

```
❌ 解密失败：不支持的状态或文件已损坏
```

**解决**: 密钥错误或文件损坏，确认密钥正确性

## 技术细节

### 加密流程

```
JSON 字符串
    ↓
AES-256-GCM 加密 (密钥 + 随机 IV)
    ↓
IV(12) + AuthTag(16) + 密文
    ↓
Base64 编码
    ↓
*.sc 文件
```

### 解密流程

```
*.sc 文件
    ↓
Base64 解码
    ↓
分离 IV + AuthTag + 密文
    ↓
AES-256-GCM 解密 (密钥 + IV + AuthTag 验证)
    ↓
JSON 字符串
```
